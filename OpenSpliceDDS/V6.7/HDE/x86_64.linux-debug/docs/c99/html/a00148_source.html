11 The Domain Participant {#DCPS_Modules_DomainParticipant}
12 =======================
13 The DomainParticipant represents the participation of the application on
14 a communication plane that isolates applications running on the same
15 set of physical computers from each other. A domain establishes a virtual
16 network linking all applications that share the same domainId and isolating
17 them from applications running on different domains. In this way, several
18 independent distributed applications can coexist in the same physical
19 network without interfering, or even being aware of each other.
20 
21 The Domain Participant has several roles:
22 
23 - It acts as the container for all other \ref DCPS_Modules_Entity "Entities"
24 - Applications that want to join a **Domain** must create a **DomainParticipant**
25 - A DomainParticipant acts as a factory for **DomainEntities** i.e. entities that either define, read or write information into the domain.
26 
27 The DomainParticipant itself is also a specialisation of the Entity (Infrastructure module) class:
28 - It has QoSPolicies that define its behaviour
29 - It has a StatusCondition that keeps track of its communication status
30 - It has a Listener that can handle the events of all embedded DomainEntities that did not handle those events themselves
