37 The purpose of this QoS is to avoid delivering "stale" data to the application.
38 Each data sample written by the \ref DCPS_Modules_Publication_DataWriter "DataWriter" has an associated ‘expiration time’ beyond which the data should not be
39 delivered to any application. Once the sample expires, the data will be removed from the DataReader caches as well as
40 from the transient and persistent information caches.
41 The ‘expiration time’ of each sample is computed by adding the duration specified by the LIFESPAN QoS to the source
42 timestamp.
43 This QoS relies on the sender and receiving applications having their clocks sufficiently synchronized. If this is not the
44 case and the Service can detect it, the DataReader is allowed to use the reception timestamp instead of the source
45 timestamp in its computation of the ‘expiration time.’
