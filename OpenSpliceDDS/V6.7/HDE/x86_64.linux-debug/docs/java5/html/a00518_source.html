1 The DataReader {#DCPS_Modules_Subscription_DataReader}
2 ===============
3 
4 The DataReader is responsible for reading **typed** sample and sample information from the middleware and is equivalent to the DataWriter, the DataReader and DataWriter provide the bulk of Listeners, conditions and QoS available in DDS as they're the intermediaries between the middleware and the application.
5 
6 The DataReader is an abstract class, that is always sub-classed by a typed specialisation, which is generated by the pre-processor:
7 - For a type **Foo** the preprocessor generates a **FooDataReader**
8  - This typed subclass contains all the functions or methods that pass parameters of type Foo
9  - *This was, type-safety is ensured by the compiler*
10  - Since the Subscriber is type-independent, it can not return a typed DataReader: the result will have to be down-casted to its proper type by the application
