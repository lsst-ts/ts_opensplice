39 This QosPolicy is used to set the key list of a DataReaderView. A DataReaderView
40 allows a different view, defined by this key list, on the data set of the \ref DCPS_Modules_Subscription_DataReader "DataReader"
41 from which it is created.
42 
43 
44 Operations that operate on instances or instance handles, such as
45 lookup_instance or get_key_value, respect the alternative key-list and work
46 as expected. However, since the mapping of writer instances to reader instances is
47 no longer trivial (one writer instance may now map to more than one matching
48 reader instance and vice versa), a writer instance will no longer be able to fully
49 determine the lifecycle of its matching reader instance, nor the value its
50 view_state and instance_state.
51 
52 
53 In fact, the view sample will always copy the view_state and instance_state
54 values from the reader sample to which it is slaved. If both samples preserve a 1 – 1
55 correspondence with respect to their originating instances (this may sometimes be
56 the case even when an alternative keylist is provided, i.e. when one reader instance
57 never maps to more than one view instance and vice versa) then the resulting
58 instance_state and view_state still have a valid semantical meaning. If this
59 1 – 1 correspondence cannot be guaranteed, the resulting
60 instance_state and view_state are semantically meaningless and should not be used to derive any
61 conclusion regarding the lifecycle of a view instance.
62 
63 
64 By default, the ViewKeyQosPolicy is disabled because use_key_list is set to FALSE.
65 
66 
67 This QosPolicy is applicable to a DataReaderView only, and cannot be changed
68 after the DataReaderView is created.
