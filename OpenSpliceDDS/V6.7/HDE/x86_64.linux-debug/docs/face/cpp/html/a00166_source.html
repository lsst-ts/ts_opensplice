38 This QosPolicy is similar to the \ref DCPS_QoS_Lifespan "LifespanQosPolicy" (applicable to \ref DCPS_Modules_TopicDefinition "Topic" and
39 \ref DCPS_Modules_Publication_DataWriter "DataWriter"), but limited to the \ref DCPS_Modules_Subscription_DataReader "DataReader" on which the QosPolicy is applied. The
40 data is automatically removed from the \ref DCPS_Modules_Subscription_DataReader "DataReader" if it has not been taken yet after
41 the lifespan duration expires. The duration of the ReaderLifespan is added to the
42 insertion time of the data in the \ref DCPS_Modules_Subscription_DataReader "DataReader" to determine the expiry time.

45 When both the ReaderLifespanQosPolicy and a \ref DCPS_Modules_Publication_DataWriter "DataWriter"â€™s \ref DCPS_QoS_Lifespan "LifespanQosPolicy" are applied to the same data,
46 only the earliest expiry time is taken into account.

48 
49 By default, the ReaderLifespanQosPolicy is not used and use_lifespan is FALSE.
50 The duration is set to DURATION_INFINITE.

53 This QosPolicy is applicable to a \ref DCPS_Modules_Subscription_DataReader "DataReader" only, and is mutable even when the
54 \ref DCPS_Modules_Subscription_DataReader "DataReader" is already enabled. If modified, the new setting will only be applied to
55 samples that are received after the modification took place.
