6 DCPS Examples {#isocpp_examples_dcpslist}
7 =============
8 
9 The examples source code can be found in the examples folder of your OpenSplice installation.
10 The descriptions below summarize what each example demonstrates. Towards the bottom of the page
11 you will find instructions on how to build and run the examples.
12 
13 Tutorial {#tutorial}
14 --------
15 - This example is a good starting point as it provides an overview of how to use
16 the most commonly used features of the API to make a message board application.
17 
18 HelloWorld {#helloworld}
19 ----------
20 - This is a simple publisher and subscriber example. One sample is published, the
21 sample is then read. Some non-default QoS are used to guarantee delivery and to
22 allow the publisher to be optionally run before the subscriber.
23 
24 ContentFilteredTopic {#contentfilteredtopic}
25 --------------------
26 - In this example a ContentFilteredTopic is used to filter the messages received by
27 the subscriber to be only those with a ticket equal to the one supplied as a parameter
28 to the program. Some non-default QoS are used to guarantee delivery.
29 
30 Durability {#durability}
31 ----------
32 - This example demonstrates the two kinds of durability, transient and persistent
33 and allows the user to specify which durability kind is used.
34 
35 Listener {#listener}
36 --------
37 - In this example a listener is registered on the datareader of the subscriber which
38 listens for the receipt of a message from the publisher and handles it. Some
39 non-default QoS are used to guarantee delivery and to allow the publisher to be
40 optionally run before the subscriber.
41 
42 Lifecycle {#lifecycle}
43 ---------
44 - This is example demonstrates the changes in Lifecycle states when a DataWriter is
45 reated, sends samples or is deleted.
46 
47 Ownership {#ownership}
48 ---------
49 - This example demonstrates the Ownership QoS. The subscriber will read messages sent
50 by the publishers and display those with the highest ownership strength.
51 
52 QueryCondition {#querycondition}
53 --------------
54 - This is a reasonably simple case publisher and subscriber example. One sample is
55 published, the sample is then read. Some non-default QoS are used to guarantee
56 delivery and to allow the publisher to be optionally run before the subscriber.
57 
58 RoundTrip {#roundtrip}
59 ---------
60 - The RoundTrip example consists of a Ping and a Pong application. Ping sends samples
61 to Pong by writing to the Ping partition which Pong subscribes to. Pong them sends
62 them back to Ping by writing on the Pong partition which Ping subscribes to. Ping
63 measures the amount of time taken to write and read each sample as well as the total
64 round trip time to send a sample to Pong and receive it back.
65 
66 Throughput {#throughput}
67 ----------
68 - The Throughput example measures data throughput in bytes per second. The publisher
69 allows you to specify a payload size in bytes as well as allowing you to specify
70 whether to send data in bursts. The publisher will continue to send data forever
71 unless a time out is specified. The subscriber will receive data and output the
72 total amount received and the data rate in bytes per second. It will also indicate
73 if any samples were received out of order. A maximum number of cycles can be
74 specified and once this has been reached the subscriber will terminate and output
75 totals and averages.
76 
77 WaitSet {#waitset}
78 -------
79 - This example demonstrates how a WaitSet can be used to wait on certain conditions
80 which will then trigger a corresponding handler functor to perform any actions
81 required such as reading data.

85 Building in a POSIX environment with make {#isocpp_examples_posixbuild}
86 =========================================
87 
88 In the following sections `$OSPL_HOME` identifies the installation directory of
89 OpenSplice. *Note* it is not however required that this be exported as a variable
90 in the shell environment.
91 
92 Building a single example
93 -------------------------
94 
95 Change directory to the example source directory and `make`. E.g.:
96 
97  $ cd $OSPL_HOME/examples/dcps/HelloWorld/isocpp
98  $ make
99 
100 Building all examples
101 ---------------------
102 
103 The top-level makefile `Makefile.DCPS_ISO_CPlusPlus` can be used to build all
104 the examples.
105 
106  $ cd $OSPL_HOME/examples
107  $ make -f Makefile.DCPS_ISO_CPlusPlus
108 
109 Building on Windows with Visual Studio {#isocpp_examples_winbuild}
110 ======================================
111 
112 A link is installed on the Windows Start Menu in the *Examples* Folder under
113 the OpenSplice HDE folder. Alternately: navigate to the `\examples` folder in
114 the OpenSplice HDE installation with Windows Explorer or the Visual Studio
115 `file -> open` dialogue and open the solution file `DCPS_ISO_CPlusPlus.sln`.

117 Running examples {#isocpp_examples_running}
118 ================
119 
120 The environment property `OSPL_URI` can set to indicate the location of a local
121 domain configuration file. If this variable is not set, or is set to point to a
122 file that specifies a _shared memory_ domain configuration, the command
123 `ospl start` must be used to start the domain before an example can be run.
124 
125 The default configuration file installed by the distribution uses _single process_
126 (i.e. heap memory) mode so this is not required.
127 
128 POSIX: {#runposix}
129 ------
130 
131 In the following examples `$OSPL_HOME` identifies the installation directory of
132 OpenSplice. *Note* it is not however required that this be exported as a variable
133 in the shell environment.
134 
135 A script `release.com` is provided. It exports a value of `OSPL_URI` to indicate
136 the file `$OSPL_HOME/etc/config/ospl.xml` and also makes the OpenSplice libraries
137 available on the library load path.
138 
139 To run an example in _single process_ mode:
140 
141  $ cd $OSPL_HOME
142  $ ./release.com
143  $ cd examples/isocpp/HelloWorld/isocpp
144  $ subscriber &
145  $ publisher
146 
147 To run an exmaple in _shared memory_ mode you might do:
148 
149  $ cd $OSPL_HOME
150  $ ./release.com
151  $ OSPL_URI=file://$OSPL_HOME/etc/config/ospl_shmem_ddsi.xml
152  $ export OSPL_URI
153  $ cd examples/isocpp/HelloWorld/isocpp
154  $ ospl start
155  $ subscriber &
156  $ publisher
157  $ ospl stop
158 
159 Windows: {#runwindows}
160 --------
161 
162 In the following examples `%%OSPL_HOME%` identifies the installation directory of
163 OpenSplice. *Note* it is not however required that this be set as a variable
164 in the console environment.
165 
166 A script `release.bat` is provided. It exports a value of `OSPL_URI` to indicate
167 the file `%%OSPL_HOME%\etc\config\ospl.xml` and also makes the OpenSplice
168 executables and dynamic-link libraries available on the `PATH`.
169 
170 A shortcut within the OpenSplice HDE folder on the Start Menu: **OpenSplice DDS
171 command prompt** creates a `cmd.exe` console session with this environment ready
172 set.
173 
174 To run an example in _single process_ mode:
175 
176  "<<< OpenSplice HDE Release V6.3.0 For x86_64.win64, Date 2013-03-27 >>>"
177  C:\ospl\V6.3.0\HDE\x86_64.win64>cd examples\isocpp\HelloWorld\isocpp
178  C:\ospl\V6.3.0\HDE\x86_64.win64\examples\isocpp\HelloWorld\isocpp>start subscriber
179  C:\ospl\V6.3.0\HDE\x86_64.win64\examples\isocpp\HelloWorld\isocpp>start publisher
180 
181 To run an example in _shared memory_ mode you might do:
182 
183  "<<< OpenSplice HDE Release V6.3.0 For x86_64.win64, Date 2013-03-27 >>>"
184  C:\ospl\V6.3.0\HDE\x86_64.win64>set OSPL_URI=file://%OSPL_HOME%\etc\config\ospl_shmem_ddsi.xml
185  C:\ospl\V6.3.0\HDE\x86_64.win64>cd examples\isocpp\HelloWorld\isocpp
186  C:\ospl\V6.3.0\HDE\x86_64.win64\examples\isocpp\HelloWorld\isocpp>ospl start
187  C:\ospl\V6.3.0\HDE\x86_64.win64\examples\isocpp\HelloWorld\isocpp>start subscriber
188  C:\ospl\V6.3.0\HDE\x86_64.win64\examples\isocpp\HelloWorld\isocpp>start publisher
189  C:\ospl\V6.3.0\HDE\x86_64.win64\examples\isocpp\HelloWorld\isocpp>ospl stop
