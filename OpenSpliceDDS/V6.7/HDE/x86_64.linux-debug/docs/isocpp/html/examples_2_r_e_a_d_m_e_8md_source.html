6 DCPS Examples {#isocpp_examples_dcpslist}

9 \subpage examplesdcpsTutorialisocpp
10 - This example is a good starting point as it provides an overview of how to use
11 the most commonly used features of the API to make a message board application.
12 
13 \subpage examplesdcpsHelloWorldisocpp
14 - This is a simple publisher and subscriber example. One sample is published, the
15 sample is then read. Some non-default QoS are used to guarantee delivery and to
16 allow the publisher to be optionally run before the subscriber.
17 
18 \subpage examplesdcpsContentFilteredTopicisocpp
19 - In this example a ContentFilteredTopic is used to filter the messages received by
20 the subscriber to be only those with a ticket equal to the one supplied as a parameter
21 to the program. Some non-default QoS are used to guarantee delivery.
22 
23 \subpage examplesdcpsDurabilityisocpp
24 - This example demonstrates the two kinds of durability, transient and persistent
25 and allows the user to specify which durability kind is used.
26 
27 \subpage examplesdcpsListenerisocpp
28 - In this example a listener is registered on the datareader of the subscriber which
29 listens for the receipt of a message from the publisher and handles it. Some
30 non-default QoS are used to guarantee delivery and to allow the publisher to be
31 optionally run before the subscriber.
32 
33 \subpage examplesdcpsLifecycleisocpp
34 - This is example demonstrates the changes in Lifecycle states when a DataWriter is
35 reated, sends samples or is deleted.
36 
37 \subpage examplesdcpsOwnershipisocpp
38 - This example demonstrates the Ownership QoS. The subscriber will read messages sent
39 by the publishers and display those with the highest ownership strength.
40 
41 \subpage examplesdcpsQueryConditionisocpp
42 - This is a reasonably simple case publisher and subscriber example. One sample is
43 published, the sample is then read. Some non-default QoS are used to guarantee
44 delivery and to allow the publisher to be optionally run before the subscriber.
45 
46 \subpage examplesdcpsRoundTripisocpp
47 - The RoundTrip example consists of a Ping and a Pong application. Ping sends samples
48 to Pong by writing to the Ping partition which Pong subscribes to. Pong them sends
49 them back to Ping by writing on the Pong partition which Ping subscribes to. Ping
50 measures the amount of time taken to write and read each sample as well as the total
51 round trip time to send a sample to Pong and receive it back.
52 
53 \subpage examplesdcpsThroughputisocpp
54 - The Throughput example measures data throughput in bytes per second. The publisher
55 allows you to specify a payload size in bytes as well as allowing you to specify
56 whether to send data in bursts. The publisher will continue to send data forever
57 unless a time out is specified. The subscriber will receive data and output the
58 total amount received and the data rate in bytes per second. It will also indicate
59 if any samples were received out of order. A maximum number of cycles can be
60 specified and once this has been reached the subscriber will terminate and output
61 totals and averages.
62 
63 \subpage examplesdcpsWaitSetisocpp

65 which will then trigger a corresponding handler functor to perform any actions
66 required such as reading data.

70 Building in a POSIX environment with make {#isocpp_examples_posixbuild}
71 =========================================
72 
73 In the following sections `$OSPL_HOME` identifies the installation directory of
74 OpenSplice. *Note* it is not however required that this be exported as a variable
75 in the shell environment.
76 
77 Building a single example
78 -------------------------
79 
80 Change directory to the example source directory and `make`. E.g.:
81 
82  $ cd $OSPL_HOME/examples/dcps/HelloWorld/isocpp
83  $ make
84 
85 Building all examples
86 ---------------------
87 
88 The top-level makefile `Makefile.DCPS_ISO_CPlusPlus` can be used to build all
89 the examples.
90 
91  $ cd $OSPL_HOME/examples
92  $ make -f Makefile.DCPS_ISO_CPlusPlus

94 Building on Windows with Visual Studio {#isocpp_examples_winbuild}
95 ======================================
96 
97 A link is installed on the Windows Start Menu in the *Examples* Folder under
98 the OpenSplice HDE folder. Alternately: navigate to the `\examples` folder in
99 the OpenSplice HDE installation with Windows Explorer or the Visual Studio
100 `file -> open` dialogue and open the solution file `DCPS_ISO_CPlusPlus.sln`.

102 Running examples {#isocpp_examples_running}
103 ================
104 
105 The environment property `OSPL_URI` can set to indicate the location of a local
106 domain configuration file. If this variable is not set, or is set to point to a
107 file that specifies a _shared memory_ domain configuration, the command
108 `ospl start` must be used to start the domain before an example can be run.
109 
110 The default configuration file installed by the distribution uses _single process_
111 (i.e. heap memory) mode so this is not required.
112 
113 POSIX:
114 ------
115 
116 In the following examples `$OSPL_HOME` identifies the installation directory of
117 OpenSplice. *Note* it is not however required that this be exported as a variable
118 in the shell environment.
119 
120 A script `release.com` is provided. It exports a value of `OSPL_URI` to indicate
121 the file `$OSPL_HOME/etc/config/ospl.xml` and also makes the OpenSplice libaries
122 available on the library load path.
123 
124 To run an example in _single process_ mode:
125 
126  $ cd $OSPL_HOME
127  $ ./release.com
128  $ cd examples/isocpp/HelloWorld/isocpp
129  $ subscriber &
130  $ publisher
131 
132 To run an exmaple in _shared memory_ mode you might do:
133 
134  $ cd $OSPL_HOME
135  $ ./release.com
136  $ OSPL_URI=file://$OSPL_HOME/etc/config/ospl_shmem_ddsi.xml
137  $ export OSPL_URI
138  $ cd examples/isocpp/HelloWorld/isocpp
139  $ ospl start
140  $ subscriber &
141  $ publisher
142  $ ospl stop
143 
144 Windows:
145 --------
146 
147 In the following examples `%%OSPL_HOME%` identifies the installation directory of
148 OpenSplice. *Note* it is not however required that this be set as a variable
149 in the console environment.
150 
151 A script `release.bat` is provided. It exports a value of `OSPL_URI` to indicate
152 the file `%%OSPL_HOME%\etc\config\ospl.xml` and also makes the OpenSplice
153 executables and dynamic-link libraries available on the `PATH`.
154 
155 A shortcut within the OpenSplice HDE folder on the Start Menu: **OpenSplice DDS
156 command prompt** creates a `cmd.exe` console session with this environment ready
157 set.
158 
159 To run an example in _single process_ mode:

164  C:\ospl\V6.3.0\HDE\x86_64.win64\examples\isocpp\HelloWorld\isocpp>start publisher
